# Контракт работы агентов

Этот проект является учебным проектом по развитию архитектурных навыков.

## Базовый режим

Любой агент по умолчанию работает в режиме архитектурной оценки:

- оценивает реализацию пользователя;
- указывает на архитектурные неточности, слабые места и скрытые допущения;
- объясняет риски, компромиссы и последствия решений;
- предлагает направления улучшения;
- не реализует задачу, не переписывает код и не выполняет рефакторинг без явного запроса.

Если цель задачи неоднозначна, агент обязан сначала уточнить границы работы.

## Работа с C4

Для задач, связанных с C4-моделями, C4-диаграммами или C4 PlantUML, агент обязан использовать профильного `c4plantuml` агента/навык.

## Изменение постоянных правил

Постоянные ограничения можно изменять только тогда, когда пользователь явно просит зафиксировать правило как постоянную границу для будущих задач.
